# Team Phynx Website Requirements

## Project

Create an English-first static Hugo website for Team Phynx at `teamphynx.github.io`.

The website presents **Soulior**, an early-stage 2D action game created by **Lazy Cat** under the **Team Phynx** identity.

The site should show game details and publish 2-week sprint devlogs to build long-term trust, community interest, and future crowdfunding readiness.

## Identity

- Project name: Phynx
- Game name: Soulior
- Developer alias: Lazy Cat
- Team/studio identity: Team Phynx
- Main identity: Team Phynx, created by Lazy Cat
- Main CTA: Follow development on GitHub

## Audience

Primary audience:

- Indie game players
- Other game developers
- Future community members

## Tone

The site should feel:

- serious
- honest
- dark
- atmospheric
- not overhyped
- realistic about early development

Main message:

> Soulior is still early, but the project is real, active, and worth following.

## Visual Style

Use a dark Japanese dark-action / soul-world inspired visual direction.

Colors:

- Background: #0B0B0D
- Surface/Card: #151518
- Text: #F2F2F2
- Muted Text: #A0A0A0
- Accent Red: #B11226
- Accent Red Hover: #E01B3C
- Border: #2A2A2E

Typography:

- Modern sans-serif
- Readable body text
- Sharp, minimal layout

## Devlog

Devlogs are published every 2-week sprint.

Create:

1. Devlog #0 — Starting Soulior
2. Sprint devlog template

Devlog content should include:

- Sprint Goal
- What I Built
- What Changed in Soulior
- Technical Notes
- Design Notes
- Problems
- What I Learned
- Prototype Media
- Next Sprint Goal

Every prototype media section must include this note:

> Note: These screenshots/GIFs are from an early prototype. Visuals and assets are temporary.

## Features V1

Include:

- Markdown posts
- Tags

- Basic SEO
- GitHub Pages deployment through GitHub Actions

Do not include yet:

- Comments
- Search
- Newsletter
- Analytics beyond GitHub traffic
- Discord community page
- Steam page
- Crowdfunding page

## Technical Requirements

- Use Hugo
- Use a customizable Hugo theme
- Prefer Blowfish if reasonable
- Host using GitHub Pages
- Use GitHub Actions deployment
- Keep content easy to maintain
- Keep the actual game repository private
- Not using emoji in website
