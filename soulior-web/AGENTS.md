# Instructions for Codex

You are helping build the Team Phynx static website.

Read `docs/website-requirements.md` before making changes.

## Goals

Build a simple Hugo website for Team Phynx / Soulior.

Prioritize:

1. Simple structure
2. Clean Hugo conventions
3. Easy Markdown content editing
4. GitHub Pages deployment
5. Dark visual style with white text and blood red accent
6. English-first content

## Do Not

- Do not expose or reference any private game repository.
- Do not add heavy JavaScript frameworks.
- Do not add comments, newsletter, search, or analytics unless requested.
- Do not overhype the game.
- Do not create fake screenshots or fake gameplay claims.
- Do not use complex architecture if a simple Hugo solution works.

## Content Style

Use simple, clear English.

Tone:

- honest
- serious
- indie
- early-development friendly
- not corporate
- not too dramatic

## Review Requirements

After each task, explain:

1. What files changed
2. How to run the site locally
3. Any remaining manual steps
